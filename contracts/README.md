# Recurra Contracts

Foundry project: `SubscriptionRegistry` (complete), `SubscriptionVault` + `PaymentExecutor` (M2).

```shell
forge build   # compile
forge test    # run the suite
```

## Local dev loop (the M1 backend harness)

Terminal 1 — a local chain:

```shell
anvil
```

Terminal 2 — deploy the Registry (uses anvil's prefunded account #0):

```shell
PRIVATE_KEY=0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80 \
forge script script/Deploy.s.sol --rpc-url http://127.0.0.1:8545 --broadcast
```

The script prints the Registry address. On a fresh anvil it is deterministic
(same key, nonce 0), so the backend `.env` can be set once:

```
ARBITRUM_RPC=http://127.0.0.1:8545
SUBSCRIPTION_REGISTRY_ADDRESS=0x5FbDB2315678afecb367f032d93F642f64180aa3
```

Restarting anvil wipes the chain; rerun the deploy and the Registry lands at
the same address.

## Seeding test data

Handy for exercising backend reads (addresses/keys are anvil defaults):

```shell
# merchant (account #1) creates "10 USDC / 30 days" — token address is a placeholder
cast send 0x5FbDB2315678afecb367f032d93F642f64180aa3 \
  "createPlan(address,uint256,uint256)" \
  0x0000000000000000000000000000000000000001 10000000 2592000 \
  --private-key 0x59c6995e998f97a5a0044966f0945389dc9e86dae88c7a8412f4603b6b78690d \
  --rpc-url http://127.0.0.1:8545

# subscriber (account #2) subscribes to plan 1
cast send 0x5FbDB2315678afecb367f032d93F642f64180aa3 \
  "subscribe(uint256)" 1 \
  --private-key 0x5de4111afa1a4b94908f83103eb1f1706367c2e68ca870fc3fb9a804cdab365a \
  --rpc-url http://127.0.0.1:8545
```

From M3, deployed addresses live in `deployments.md` at the repo root — env
files only, never hardcoded.
